Using an Encrypted Stratis Pool with Network-Bound Disk Encryption (NBDE)
Welcome to this guide to using an encrypted pool
Stratis - is a local storage management service that makes it easy to manage pools from physical storage devices.
   _____

There are several concepts involved here:
pool: This is the main component of Stratis. It consists of one or more block devices, and its total size is equal to the sum of the block device sizes.

blockdev: These are block devices on Stratus. The following block devices are supported here: 
1 hard drives and solid state drives 
2 LVM logical volumes
3 MD RAID
4 iSCSI
5 DM Multipath
6 SUITE

File system: A pool can contain one or more file systems formatted in XFS and used for storing files. Usually file systems have thin software. In other words, they do not have a fixed total size, and their size grows as data is stored on them.
There exist many functions related to the Stratis pool, including:
  Pool-based management
  Snapshots of the file system
  Monitoring
  Thin provisioning
  Tiering
Startis allows you to create both encrypted and unencrypted pools. Encrypted pools are created on Stratis to increase security. When creating a pool of this type, the core keychain is used as the main encryption mechanism. After restarting the system, to access the pool, you must provide the created kernel keychain. In this guide, you will gain the necessary knowledge on how to use the Stratus encrypted pool with Network Bound Disk Encryption (NBDE). A Stratis pool can be created using one or more block devices. In this guide, we use 3 secondary disks connected to the system. Start by deleting all partition tables, file systems, and any RAID signatures on the device.
sudo wipefs --all block-device1block-devic
Replace block-device 1 and block-device 2 with device names.
wipefs --all /dev/sdb /dev/sdc /dev/sdd
After clearing the disk(s), proceed to obtaining a set of keys for encryption.
	stratis key set --capture-key pool1key
	Enter key data followed by the return key: Enter Key and press Enter
Now you should have a kernel keychain created. It will be used to access the pool after subsequent reboots. Check with the command:
	stratis key list
	Key Description
	pool1key
Create an encrypted pool using the command with the syntax below:
	stratis pool create 
	--key-desc <key-description> <pool_name> 
	block-device1 block-device2 block-device-n
After creating it, check it with the command:
	     stratis pool list
	Name                       Total Physical   Properties                                   UUID
	pool1   29.95 GiB / 45.64 MiB / 29.91 GiB      ~Ca, Cr   be1d8a96-03f8-41a4-9d63-09baa1c697d8
We have a pool named pool1, which has a capacity of 30 GB. This is the sum of the capacity of 3 disks.
After creating a pool, you need to create a file system on it to be able to write data to the pool. This can be done using the command with the syntax below:
stratis fs create <poolname> <filesys
example:
stratis filesystem create pool1 filesystem1
Check the creation using the command:
			 stratis fs list
	Pool Name   Name          Used      Created             Device                           UUID
	pool1       filesystem1   545 MiB   Jul 07 2022 13:12   /dev/stratis/pool1/filesystem1   283f8f78-0b33-4a35-a2d0-e9cd6b85d124
To get a detailed output, use the command below:
lsblk
After restarting the system, you will not be able to access the pool, to access it, you must provide the created kernel key. First recreate the key using the same key description.
stratis key set --capture-key pool1key
Now unlock the pool with the key:
stra
Get access to the pool:
	stratis pool list
	Name                        Total Physical   Properties                                   UUID
	pool1   29.95 GiB / 590.65 MiB / 29.38 GiB      ~Ca, Cr   be1d8a96-03f8-41a4-9d63-09baa1c697d8
This is quite convenient, but requires memorizing a bunch of keys. This can be simplified using NBDE as shown below.
To be able to bind an encrypted Stratis pool to NBDE, you need a Tang server. This Tang server will help unlock the encrypted pool without having to provide a description of the kernel keychain as described above. First install the Tang server:
yum install tang
Select any unused port and bind the service to it. For example, port 7500 as shown below:
semanage port -a -t tangd_port_t -p tcp 7500
Allow the specified port on the firewall:
firewall-cmd --add
Enable the tangd service:
systemctl enable tangd.socket
Create an override configuration file in /etc/systemd/system/tangd.socket.d/ using the command:
syst
Change the default port 80 to the new port as shown below:
	[Socket]
	ListenStream=
	ListenStream=7500
Restart the daemon and restart the service:
sys
Check if the service is bound to the new port:
	systemctl show tangd.socket -p Listen
	Listen=[::]:7500 (Stream)
Get a thumbprint of the tang server:
	tang-show-keys 7500
	Mbe0_qFvUj7GRHXUp6g6yEQhixU
Now bind the encrypted Stratis pool to NBDE using the command with the syntax below:
stratis pool bind nbde <pool_name> <tang-server> --t
Replace poll_name and tang-server accordingly.
To check if we can unlock the Stratis pool using NBDE, reboot the system.
reboot now
After a successful system reboot, switch to the root user.
su
	stratis pool list
	Name                        Total Physical   Properties                                   UUID
In addition to unlocking the pool with a key, you can unlock it using NBDE (spike), as shown below:
stratis p
You can agree that you no longer need to provide a keychain to access the pool.
Decoupling the Stratis pool from additional ciphers leaves only the main cipher of the core keychain in effect. Additional encryption can be associated with NBDE or TPM. To remove this binding, use the command with the syntax below:
stratis pool unbind clevis <pool_name></pool_name>
example:
stratis pool unbind clevis pool1
After that, access to the pool can only be obtained using the primary encryption key of the kernel
To use the Stratis file system, it must be mounted. First create a mount point:
mkdir /mnt/pool_1
You can perform a temporary file system mount using the command:
mount /dev/stratis
To mount a file system permanently, start by defining the UUID of the file systems:
	lsblk --output=UUID /dev/stratis/pool1/filesystem1
	UUID
	65a09ba7-1053-424c-9e04-d21c5c090249
Now edit the /etc/fstab file and add the file system.
	sudo vi /etc/fstab
	UUID=65a09ba7-1053-424c-9e04-d21c5c090249 /mnt/pool_1 xfs defaults,x-systemd.requires=stratisd.service 0 0
Don't forget to replace the UUID and mount point. Restart the system daemon:
systemctl daemon-reload
After that, the file system will be able to survive a system reboot.

How to determine which WAF protects a website or application?
Web Application Firewall filters and controls the traffic passing through it. Its main function is to ensure the protection of web applications from the main disadvantages.
WAF differs from a traditional firewall in that it can filter the contents of specific online applications, while traditional firewalls act as a protective barrier between servers.

WHATWAF
Whatwaf is a security tool for web application information and detecting the presence of any WAF. This tool is useful for determining whether a web application is protected by WAF during a security assessment. If this is the case, circumvention and prevention strategies may be useful when further testing or operating the web application.

Wafw00f
The most well-known tool for detecting web application firewall is Wafw00f. Wafw00f sends an HTTP request to the web application firewall to identify it. If sending an HTTP request fails, wafw00f makes a malicious HTTP request. If sending a malicious HTTP request fails, wafw00f examines previous HTTP requests and uses a simple algorithm to determine if the web application firewall is responding to our attacks.

This is a penetration tester tool that allows the auditor to intercept SSH connections. The tool is based on the OpenSSH v7.5p1 source code, which is patched in a special way, which turns it into a proxy between the victim and the SSH server to which she is trying to connect; all passwords transmitted in plain text and sessions are intercepted and saved to the attacker's disk.
Of course, the victim's SSH client will complain that the server key has changed. But since in 99.99999% of cases this is caused by legitimate actions (reinstalling the OS, configuration changes, etc.), many/most users ignore this warning and continue.
note: It is recommended to run the modified sshd_mitm only in the virtual machine container! The changes made in OpenSSH affect critical areas of the code without regard to security. It's not hard to imagine that these changes have opened up serious vulnerabilities.
-h
The program has no options, files are used to start and stop the attack start.sh and stop.sh . Script JoesAwesomeSSHMITMVictimFinder.py it is used to search for targets before an attack. That is, this script is looking for computers with SSH connections.
Launch examples
JoesAwesomeSSHMITMVictimFinder script.py makes finding targets on the local network very simple. It performs an ARP attack spoofing IP blocks for a short time and sniff traffic in search of SSH connections. IP addresses are not attacked simultaneously, but in small blocks for a short time — all blocks are processed sequentially, when the script reaches the last one, then everything starts over. It will report all outgoing SSH connections coming from devices on the LAN. By default JoesAwesomeSSHMITMVictimFinder.py it will perform ARP spoofing and sniff only 5 IP addresses at a time for 20 seconds before moving on to the next block of 5 addresses. These parameters can be configured based on the following compromise: the more IP is spoofed (attacked) at a time, the higher the chance that you will catch an outgoing SSH connection, but also the load on your network interface increases, which is rarely designed for such intensive work on home computers. Under too much load, your interface will start dropping frames, which will lead to denial-of-service (network problems) and greatly increase suspicion (which is bad). This default value should not lead to problems in most cases, although it will lead to a longer search for goals. For low-load networks, the block size can be safely increased.

Nmap ("Network Mapper") is an open source utility for network exploration and security verification. Many system administrators and network administrators also find it useful for tasks such as network inventory, managing the schedule of update services and monitoring uptime of hosts or services. Nmap uses raw IP packets in an innovative way to determine which hosts are available on the network, which services (application name and version) these hosts suggest which operating systems (and OS versions) are running there, what types of packet filters/firewalls are used, and dozens of other characteristics. It was created to quickly scan large networks, but it works and works fine with single hosts. Nmap runs on all popular operating systems, and official executable packages are available for Linux, Windows, and Mac OS X. In addition to the classic version of the command-line Nmap, the Nmap suite includes an advanced graphical interface and results viewer (Zenmap), a flexible data transfer, redirection and debugging tool (Ncat), a utility for comparing scan results (Ndiff), and a packet generation and response analysis tool (Nping). He has even been featured in twelve films, including The Matrix Reboot, Die Hard 4, Girl With the Dragon Tattoo and the Bourne Ultimatum.

Nmap (“Network Mapper”) is an open source utility for network exploration and security verification. It was designed to quickly scan large networks, although it copes well with single targets. Nmap uses raw IP packets in original ways to determine which hosts are available on the network, which services (application name and version) they suggest which operating systems (and OS versions) they use, what types of packet filters/firewalls are used and dozens more other characteristics. While Nmap is commonly used for security checks, many network and system administrators find it useful for common tasks such as monitoring network structure, managing service startup schedules, and keeping track of host or service hours. The output of Nmap is a list of scanned targets with additional information for each, depending on the specified options. The key information is the “table of important ports". This table contains the port number, protocol, service name, and status. The state can have the value open (open), filtered (filtered), closed (closed) or unfiltered (not filtered). Open means that the application on the target machine is ready to establish a connection/accept packets on this port. Filtered means that a firewall, network filter, or some other interference in the network is blocking the port, and Nmap cannot set whether this port is open or closed. Closed ports are not associated with any application, so they can be opened at any time. The port is classified as not filtered when they respond to Nmap probes, but Nmap cannot determine if they are open or closed. Nmap reports the status of ports with combinations of open|filtered and closed|filtered when it cannot determine which of the two states describes the port. If there was a request to determine the version, the ports table may also include details about the software version. When an IP Protocol (-sO) scan is requested, Nmap provides information about supported IP protocols, not about the ports being listened to. In addition to the table of important ports, Nmap can provide further information about the targets: converted DNS names, an assumption about the operating system used, device types and MAC addresses. 	unsigned char checksum, c, *binary, *keybuf;

		return NULL;

		return NULL;

		keybuf[i] += keybuf[i-1];
  
	for (i=keylen-1; i >= 1; i--)
		keybuf[i-1] += keybuf[i];
  
	/* Output string is never larger than input string */
	binary = (unsigned char *)malloc(asciilen);
  
	/* Convert from 6-bit ASCII to 8-bit binary */
	binary[j] = 0;
	needed = 8;
	for (i=1; i < asciilen; i++) {
		c = 0xFF;
		for (k=0 ; k < 64 ; k++) {
			if (ascii[i] == SixbitToChar[k]) {
				c = k;
				break;
			}
		}
      
		if (c == 0xFF) {
			free(binary);
			return 0;
		}
      
		avail = 6;
		while (avail > 0) {
			unsigned int bits = (avail < needed) ? avail : needed;
	  
			/* Fill empty space with leftmost bits */
			binary[j] <<= bits;
			binary[j] |= (c>>(6-bits)) & Mask[bits];
			c <<= bits;
			avail -= bits;
			needed -= bits;
	  
			if (needed == 0) {
				binary[++j] = 0;
				needed = 8;
			}
		}
	}
  
	clearlen = j;
  
	/*
	** For each character from the first to n-1, subtract the
	** character to the right.
	*/
	for (i=0; i < clearlen-1; i++)
		binary[i] -= binary[i+1];

		binary[i] -= binary[i-1];
		binary[i] -= keybuf[i % keylen];
	}

	/* binary[1] = password length */
	/* binary[2] = password nibble checksum */
	/* binary[3...] = password */

		free(binary);
		return 0;       /* Invalid password */
	}

		return 0;       /* Invalid password */

	return ((char *)binary);

MAC Address Spoofing on Kali
The MAC address is a unique combination of numbers and letters with a length of 48 characters. In simple words, it is the hardware physical number of the device (server, PC, router, etc.), which is assigned to the device's network card during production. MAC spoofing is a method of changing/spoofing the MAC address of a network device. MACChanger is a small utility that allows you to change the MAC address of the network card of a computer running the Linux operating system.
MACChanger Parameters:
-h — help output.
-V — display the version.
-s — display the MAC address.
-e — do not change the manufacturer's bytes (the first three bytes).
-a — set a random MAC address of the same type.
-A — set a random MAC address of any type.
-p — reset to original, permanent hardware MAC
-r — set a random MAC address.
-l —list[=STRING] — display information about manufacturers.
-b — pretend to be a stitched address.

ifconfig

sudo ifconfig eth0 down
To replace the MAC address (random change), run the command:
sudo macchanger -A eth0
Now you need to enable the eth0 interface:
sudo ifconfig eth0 up
To check the MAC address and the status of the eth0 interface, run the command "ifconfig" again
