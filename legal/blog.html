Using an Encrypted Stratis Pool with Network-Bound Disk Encryption (NBDE)
Welcome to this guide to using an encrypted pool
Stratis - is a local storage management service that makes it easy to manage pools from physical storage devices.
   _____
  |==   | <- investigate
  |  o. |  
  |__O__|
|
There are several concepts involved here:
pool: This is the main component of Stratis. It consists of one or more block devices, and its total size is equal to the sum of the block device sizes.

blockdev: These are block devices on Stratus. The following block devices are supported here: 
1 hard drives and solid state drives 
2 LVM logical volumes
3 MD RAID
4 iSCSI
5 DM Multipath
6 SUITE

File system: A pool can contain one or more file systems formatted in XFS and used for storing files. Usually file systems have thin software. In other words, they do not have a fixed total size, and their size grows as data is stored on them.
There exist many functions related to the Stratis pool, including:
  Pool-based management
  Snapshots of the file system
  Monitoring
  Thin provisioning
  Tiering
Startis allows you to create both encrypted and unencrypted pools. Encrypted pools are created on Stratis to increase security. When creating a pool of this type, the core keychain is used as the main encryption mechanism. After restarting the system, to access the pool, you must provide the created kernel keychain. In this guide, you will gain the necessary knowledge on how to use the Stratus encrypted pool with Network Bound Disk Encryption (NBDE). Stratus allows you to create both encrypted and unencrypted pools. Encrypted pools are created on Stratis to increase security. When creating a pool of this type, the core keychain is used as the main encryption mechanism. After restarting the system, to access the pool, you must provide the created kernel keychain.

A Stratis pool can be created using one or more block devices. In this guide, we use 3 secondary disks connected to the system. Start by deleting all partition tables, file systems, and any RAID signatures on the device.
sudo wipefs --all block-device1block-devic
Replace block-device 1 and block-device 2 with device names.
wipefs --all /dev/sdb /dev/sdc /dev/sdd
After clearing the disk(s), proceed to obtaining a set of keys for encryption.
	stratis key set --capture-key pool1key
	Enter key data followed by the return key: Enter Key and press Enter
Now you should have a kernel keychain created. It will be used to access the pool after subsequent reboots. Check with the command:
	stratis key list
	Key Description
	pool1key
Create an encrypted pool using the command with the syntax below:
	stratis pool create 
	--key-desc <key-description> <pool_name> 
	block-device1 block-device2 block-device-n
After creating it, check it with the command:
	     stratis pool list
	Name                       Total Physical   Properties                                   UUID
	pool1   29.95 GiB / 45.64 MiB / 29.91 GiB      ~Ca, Cr   be1d8a96-03f8-41a4-9d63-09baa1c697d8
We have a pool named pool1, which has a capacity of 30 GB. This is the sum of the capacity of 3 disks.
After creating a pool, you need to create a file system on it to be able to write data to the pool. This can be done using the command with the syntax below:
stratis fs create <poolname> <filesys
example:
stratis filesystem create pool1 filesystem1
Check the creation using the command:
			 stratis fs list
	Pool Name   Name          Used      Created             Device                           UUID
	pool1       filesystem1   545 MiB   Jul 07 2022 13:12   /dev/stratis/pool1/filesystem1   283f8f78-0b33-4a35-a2d0-e9cd6b85d124
To get a detailed output, use the command below:
lsblk
After restarting the system, you will not be able to access the pool, to access it, you must provide the created kernel key. First recreate the key using the same key description.
stratis key set --capture-key pool1key
Now unlock the pool with the key:
stra
Get access to the pool:
	stratis pool list
	Name                        Total Physical   Properties                                   UUID
	pool1   29.95 GiB / 590.65 MiB / 29.38 GiB      ~Ca, Cr   be1d8a96-03f8-41a4-9d63-09baa1c697d8
This is quite convenient, but requires memorizing a bunch of keys. This can be simplified using NBDE as shown below.
To be able to bind an encrypted Stratis pool to NBDE, you need a Tang server. This Tang server will help unlock the encrypted pool without having to provide a description of the kernel keychain as described above. First install the Tang server:
yum install tang
Select any unused port and bind the service to it. For example, port 7500 as shown below:
semanage port -a -t tangd_port_t -p tcp 7500
Allow the specified port on the firewall:
firewall-cmd --add
Enable the tangd service:
systemctl enable tangd.socket
Create an override configuration file in /etc/systemd/system/tangd.socket.d/ using the command:
syst
Change the default port 80 to the new port as shown below:
	[Socket]
	ListenStream=
	ListenStream=7500
Restart the daemon and restart the service:
sys
Check if the service is bound to the new port:
	systemctl show tangd.socket -p Listen
	Listen=[::]:7500 (Stream)
Get a thumbprint of the tang server:
	tang-show-keys 7500
	Mbe0_qFvUj7GRHXUp6g6yEQhixU
Now bind the encrypted Stratis pool to NBDE using the command with the syntax below:
stratis pool bind nbde <pool_name> <tang-server> --t
Replace poll_name and tang-server accordingly.
To check if we can unlock the Stratis pool using NBDE, reboot the system.
reboot now
After a successful system reboot, switch to the root user.
su
	stratis pool list
	Name                        Total Physical   Properties                                   UUID
In addition to unlocking the pool with a key, you can unlock it using NBDE (spike), as shown below:
stratis p
You can agree that you no longer need to provide a keychain to access the pool.
Decoupling the Stratis pool from additional ciphers leaves only the main cipher of the core keychain in effect. Additional encryption can be associated with NBDE or TPM. To remove this binding, use the command with the syntax below:
stratis pool unbind clevis <pool_name></pool_name>
example:
stratis pool unbind clevis pool1
After that, access to the pool can only be obtained using the primary encryption key of the kernel
To use the Stratis file system, it must be mounted. First create a mount point:
mkdir /mnt/pool_1
You can perform a temporary file system mount using the command:
mount /dev/stratis
To mount a file system permanently, start by defining the UUID of the file systems:
	lsblk --output=UUID /dev/stratis/pool1/filesystem1
	UUID
	65a09ba7-1053-424c-9e04-d21c5c090249
Now edit the /etc/fstab file and add the file system.
	sudo vi /etc/fstab
	UUID=65a09ba7-1053-424c-9e04-d21c5c090249 /mnt/pool_1 xfs defaults,x-systemd.requires=stratisd.service 0 0
Don't forget to replace the UUID and mount point. Restart the system daemon:
systemctl daemon-reload
After that, the file system will be able to survive a system reboot.

WAF
How to determine which WAF protects a website or application?
Web Application Firewall filters and controls the traffic passing through it. Its main function is to ensure the protection of web applications from the main disadvantages.
WAF differs from a traditional firewall in that it can filter the contents of specific online applications, while traditional firewalls act as a protective barrier between servers.

WHATWAF
Whatwaf is a security tool for web application information and detecting the presence of any WAF. This tool is useful for determining whether a web application is protected by WAF during a security assessment. If this is the case, circumvention and prevention strategies may be useful when further testing or operating the web application.
Install-Kali:
	sudo apt install python3-pip

	git clone https://github.com/ekultek/whatwaf

	cd whatwaf

	sudo pip3 install -r requirements.txt
At startup, without having a Python version, we can easily specify an executable file:
./whatwaf --help
The WhatWaf firewall detection tool is easy to use. We just need to run the following command:
./whatwaf -u https://www.targetsite.com
We can also use the tor service to scan the WAF, but this may increase the latency. Just type (--tor) in the end of above command.
The main advantage of the WhatWaf tool is that it automatically tries to transmit the payload bypassing the detected firewall.
Wafw00f
The most well-known tool for detecting web application firewall is Wafw00f. Wafw00f sends an HTTP request to the web application firewall to identify it. If sending an HTTP request fails, wafw00f makes a malicious HTTP request. If sending a malicious HTTP request fails, wafw00f examines previous HTTP requests and uses a simple algorithm to determine if the web application firewall is responding to our attacks.

This category is still an integral part of the Top 10, but the increased availability of standardized frameworks seems to be helping.
A08:2021-Software and Data Integrity Failures is a new category for 2021, focusing on making assumptions related to software updates, critical data, and CI/CD pipelines without verifying integrity. One of the highest weighted impacts from Common Vulnerability and Exposures/Common Vulnerability Scoring System (CVE/CVSS) data mapped to the 10 CWEs in this category. Insecure Deserialization from 2017 is now a part of this larger category.
A09:2021-Security Logging and Monitoring Failures was previously Insufficient Logging & Monitoring and is added from the industry survey (#3), moving up from #10 previously. This category is expanded to include more types of failures, is challenging to test for, and isn’t well represented in the CVE/CVSS data. However, failures in this category can directly impact visibility, incident alerting, and forensics.
A10:2021-Server-Side Request Forgery is added from the Top 10 community survey (#1). The data shows a relatively low incidence rate with above average testing coverage, along with above-average ratings for Exploit and Impact potential. This category represents the scenario where the security community members are telling us this is important, even though it’s not illustrated in the data at this time.

It is designed for users with a wide range of security experience, so it is great for developers and functional testers who are not familiar with pentests.
ZAP creates a proxy server between the client and your site. While you are navigating your website, it captures all the actions, and then attacks the site with known methods.

So what is the difference between active and passive scanning?
From the point of view of the penetration test, passive scanning is a harmless test that only looks for answers and checks them for known vulnerabilities. Passive scanning does not change the site data, which makes it more secure. // Active scanning attacks the website with various known methods. Active scanning can modify data and insert malicious scripts. Therefore, it is better to test sites that are deployed on a local host in this way.

|A
Account Takeover (ATO) - hacker compromises credentials to gain access to a system, application, or user account.
Address Poisoning Attack (APA) - It is an attack that occurs by replacing the recipient's address with the victim's address. The hacker creates a transaction from his wallet, but substitutes the victim's address for the sender's wallet and puts an amount close to zero. Thus, the victim can receive a notification about the transaction, but will not know that it was not committed by her. The victim may inadvertently assume that she did this transaction herself and send the money to the fraudster's address.
Adversary-in-The-Middle (AiTM) - This is a type of cyber attack in which attackers try to take a position between two or more network devices. Using the functions of common network protocols that can determine the flow of network traffic (for example, ARP, DNS, LLMNR, etc.), attackers are able to intercept the data stream sent by the device and redirect it to their systems to collect information or perform additional actions they need.
ARP-spoofing/poisoning - a type of MITM-type network attack used in networks using the ARP protocol. It is mainly used in Ethernet networks. The attack is based on the shortcomings of the ARP protocol. The Address Resolution Protocol (ARP) is a protocol that allows network communications to reach a specific device on the network. ARP converts Internet Protocol (IP) addresses to Media Access Control (MAC) addresses and vice versa. Most often, devices use ARP to communicate with a router or gateway that allows them to connect to the Internet.
Advanced Persistent Threat (APT) - In an APT attack, a threat actor uses the most sophisticated tactics and technologies to penetrate a high-profile network. APTs aim to stay ‘under the radar’ and explore the network while remaining undetected for weeks, months, and even years. APTs are most often used by nation-state threat actors wishing to cause severe disruption and damage to the economic and political stability of a country. They can be considered the cyber equivalent of espionage ‘sleeper cells’.
Attack Vector - An Attack Vector is the collection of all vulnerable points by which an attacker can gain entry into the target system. Attack vectors include vulnerable points in technology as well as human behavior, skillfully exploited by attackers to gain access to networks. The growth of IoT devices and (Work from Home) have greatly increased the attack vector, making networks increasingly difficult to defend.
Advanced Threat Protection (ATP) - Advanced Threat Protection (ATP) are security solutions that defend against sophisticated malware or hacking attacks targeting sensitive data. Advanced Threat Protection includes both software and managed security services.
|B
Bring Your Own Filesystem (BYOF) - an attacker creates a malicious file system on his device containing a standard set of hacker tools for carrying out attacks.This file system is then loaded onto the compromised computer, providing the hacker with a pre-created set of tools to further compromise the Linux system.
Business Email Compromise (BEC) - this is a type of cyber attack in which an attacker gains access to the email account of a target organization through phishing, social engineering, or the purchase of credentials on the darknet.
Bring Your Own Vulnerable Driver (BYOVD) - a method of attack for which hackers use a harmless driver, which at the same time contains known vulnerabilities. Since such drivers have a signature, the OS allows them to be installed.
Bluesnarfing - unauthorized access to information from a wireless device via a Bluetooth connection, often between phones, desktops, laptops and PDAs (personal digital assistant).
Bluejacking - Also known as Bluetooth spam attack, it is a technique in which an attacker sends anonymous text messages using Bluetooth enabled.
Blacklist, Blocklist, Denylist - is a basic access control mechanism that allows elements such as email addresses, users, passwords, URLs, IP addresses, domain names, file hashes, etc. through the system, except those explicitly mentioned which are denied access. Bot A Bot is a program that automates actions on behalf of an agent for some other program or person and is used to carry out routine tasks. Their use for malicious purposes includes spam distribution, credentials harvesting, and the launching of DDoS attacks.
Brute Force Attack - This is a method for guessing a password (or the key used to encrypt a message) that involves systematically trying a high volume of possible combinations of characters until the correct one is found. One way to reduce the susceptibility to a Brute Force Attack is to limit the number of permitted attempts to enter a password – for example, by allowing only three failed attempts and then permitting further attempts only after 15 minutes.

|C
Cache Cramming - this is a method of tricking the browser to run cached Java code from a local disk, and not from the Internet zone, so it works with less strict permissions.
Credential stuffing - this is a method of data theft that is used by attackers to access accounts on the Internet. It is based on the use of databases of accounts that were merged into the network as a result of earlier data leaks. By the method of automatic search, special software goes through the database and finds the desired login / password combination.
Cross Site Request Forgery (CSRF) - an attack on users of web sites, in the framework of which the shortcomings of the HTTP protocol are used. When visiting a fraudulent resource, a request is secretly sent on behalf of the user to another server that carries out certain malicious activities (for example, transferring money to the account of fraudsters). To successfully carry out the attack, the victim must be authorized on the server to which the request is sent. Moreover, the request should not require confirmation from the user, since it can be ignored or forged by an attacking script. Among the applications of CSRF, the exploitation of passive XSS detected on another server is distinguished. In addition, it is also possible to send spam on behalf of the victim and change account settings on other sites (for example, a secret question for password recovery).
Clickjacking - Clickjacking involves tricking someone into clicking on one object on a web page while they think they are clicking on another. The attacker loads a transparent page over the legitimate content on the web page so that the victim thinks they are clicking on a legitimate item when they are really clicking on something on the attacker’s invisible page. This way, the attacker can hijack the victim’s click for their own purposes. Clickjacking could be used to install malware, gain access to one of the victim’s online accounts, or enable the victim’s webcam.
Cryptojacking - Cryptojacking consists of hackers using the computing power of a compromised device to generate or “mine” cryptocurrency without the owner’s knowledge. Mining can be performed either by installing a malicious program on the target computer or through various kinds of fileless malware. Sometimes attackers take over part of the computer’s processing power when a page containing a special mining script is opened. Cryptojacking has been known to occur when viewing online ads or solving a CAPTCHA.
|D
DCShadow - this is a late-stage chain of destruction attack that allows an attacker with privileged credentials to register a "fraudulent" domain controller in order to send changes to the domain through domain replication. Embedded replicated events are logged, processed, and recorded as valid domain replication. As a result, an attacker can make changes through replication in a way that is very difficult to detect.
DCSync - this is an attack that allows an attacker to simulate the behavior of a domain controller (DC) and extract password data through domain replication. The classic use of DCSync is a precursor to the Golden Ticket attack, since it can be used to get a KRBTGT hash.
DDOS - distributed denial of service attack, which is one of the most common and dangerous network attacks. As a result of the attack, the service of legitimate users, networks, systems and other resources is disrupted or completely blocked. DDoS attack consists in continuous access to the site from many computers located in different parts of the world. In most cases, these computers are infected with viruses that are managed centrally by scammers and combined into one botnet (botnet). Computers that are part of the botnet send spam, thus participating in DDoS attacks.
Dead Drop Resolver (DDR) - an attack technique in which attackers place content with embedded malicious domains or IP addresses on legitimate web services, trying to hide their intentions. The malicious code does not contain the address of the C&C server - instead, the program accesses a post published in a public service and reads from it a string of characters that at first glance seem meaningless. In fact, this is encrypted information that serves to activate the next stage of the attack.
Dynamic linker hijacking (DLH) - Attackers can execute their own malicious payloads by intercepting environment variables that the dynamic linker uses to load shared libraries. On Linux and macOS, intercepting dynamic linker variables can grant access to the victim process's memory, system/network resources, and possibly elevated privileges. This method can also avoid detection by security products because the execution is masquerading as a legitimate process.
DNS Hijacking - Attacker changes the IP address of a resource using malware and redirects the victim to a compromised server controlled by the criminal instead of the requested site. Internet service providers intercept DNS to collect statistics and display ads. In addition, DNS service providers can use traffic interception as censorship to prevent access to certain pages.
DLL Injection - a method used to run code in the address space of another process, forcing it to load a malicious DLL library. For example, embedded code can intercept system function calls or read the contents of password text fields.
Drive-By Download - during this attack, a malicious file is quietly downloaded from the browser to the target system without the victim's knowledge. The file can be downloaded through advertising, a floating frame (iframe) or a malicious script embedded in the site.
DLL Sideloading - interception of the DLL library that the program loads. Instead of simply installing the DLL in the order of the program search and then waiting for the victim application to be called, the attacker directly loads his payload by installing and then running a legitimate application that executes his payload.
Domain Shadowing - this is a method of compromise, which involves hacking the domain administrator account and creating many subdomains in it in order to bypass the blocking by lists. Malicious pages linked to these URLs are hosted on the servers of criminals and are not connected in any way with the hacking victim's web resource (there are no links to them on the pages of the main site).

Fileless Malware - Fileless Malware (FM), aka “non-malware,” or “fileless infection,” is a form of malicious computer attack that exists exclusively within the realm of volatile data storage components such as RAM, in memory processes, and service areas. This differentiates this form of malware from the classic memory-resident virus which requires some contact with non-volatile storage media, such as a hard disk drive or a thumb drive. Normally picked up following visits to malicious websites, fileless malware does not exist as a file that can be detected by standard antivirus programs. It lurks within a computer’s working memory and is exceptionally difficult to identify. However, this type of malware rarely survives a computer reboot, after which the computer should work as it did prior to infection.

The malware generates ultrasonic signals at the resonant frequencies of the MEMS gyroscope. These inaudible frequencies produce tiny mechanical vibrations in the smartphone's gyroscope, which can be demodulated into binary information.

The phishing attempt will try to encourage a recipient, for one reason or another, to enter/update personal data. Common reasons given can include “suspicious login to the account,” or “expiration of the password.”
